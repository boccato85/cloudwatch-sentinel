### Finding 1: Hardcoded Default AUTH_TOKEN

**Files:** `agent/main.go` Â· `agent/static/dashboard.js` Â· `helm/sentinel/values.yaml`
**Severity:** High | **Confidence:** 9/10 | **Category:** `hardcoded_credential`

**Description:** `AUTH_ENABLED` is now `true` by default (correct direction), but `AUTH_TOKEN` is given a static default of `sentinel-secure-token` in three simultaneous places: the Go binary fallback (`getEnv("AUTH_TOKEN", "sentinel-secure-token")`), the JS last-resort fallback in `dashboard.js:5`, and `helm/sentinel/values.yaml:54`. The default is also explicitly documented in `README.md`. Any deployment that doesn't override this value â€” including a bare `helm install sentinel ./helm/sentinel` â€” runs with a publicly known token.

**Exploit scenario:** Attacker reads the README (public repo) or views page source of `/static/dashboard.js` (a no-auth route). Sends `Authorization: Bearer sentinel-secure-token` to any `/api/*` endpoint and receives full access to cluster metrics, incidents, pod topology, and cost data.

**Fix:**
- Go: remove the string default â€” if `AUTH_ENABLED=true` and `AUTH_TOKEN` is empty, `slog.Error` + `os.Exit(1)`.
- JS: remove the `|| 'sentinel-secure-token'` fallback â€” show an error or setup prompt instead.
- Helm: replace the default value with a `{{ required "agent.auth.token must be set" .Values.agent.auth.token }}` guard.

### Finding 2: `/health` Leaks Internal Network Topology (Unauthenticated)

**File:** `agent/pkg/api/api_handlers.go:72-97`
**Severity:** Medium | **Confidence:** 9/10 | **Category:** `data_exposure`

**Description:** `AuthMiddleware` explicitly exempts `/health` from auth regardless of `AUTH_ENABLED`. The handler returns `dbErr.Error()`, `k8sErr.Error()`, and `metricsErr.Error()` verbatim in the JSON response. Go's `net` package error strings routinely contain internal cluster IPs and ports, e.g. `dial tcp 10.96.0.5:5432: connect: connection refused`. The circuit-breaker state (`DBBreakerState`) was also added to this response in this diff, adding another internal signal on the public route.

**Exploit scenario:** Attacker calls `GET /health` during any transient connectivity issue and reads the database service IP+port, the K8s API server address, and the metrics-server address â€” actionable topology for lateral movement or targeted attack scoping.

**Fix:** Replace `dbErr.Error()` with a static string (`"database unreachable"`) in the HTTP response. Log the raw error server-side only with `slog.Error`.

### Finding 3: DOMPurify Removed â€” `innerHTML` Safety Net Gone

**Files:** `agent/static/dashboard.js:1689` Â· `agent/static/dashboard.html`
**Severity:** Medium | **Confidence:** 8/10 | **Category:** `xss`

**Description:** This diff simultaneously removes the DOMPurify script import from `dashboard.html` and simplifies `drawerHTML()` to a bare `innerHTML` assignment. The `esc()` helper (a proper `createTextNode`-based encoder) is consistently applied to known string fields, but one concrete unescaped server string was identified: `entry.opportunity` inserted into `innerHTML` without `esc()`. The current backend generates this field as a numeric-derived string (`"-NNNm"`), but it is a server-controlled string path with no escaping guard. More broadly, DOMPurify was the last-resort safety net for any future field that misses an `esc()` call â€” its removal converts "hardened with belt-and-suspenders" to "correct only as long as every future contributor remembers `esc()`."

Additionally, `n.namespace` is inserted raw at lines 1510 and 1521 (Kubernetes enforces DNS-label naming so this is safe today, but is another unguarded path).

**Fix:** Restore DOMPurify or switch all drawer rendering to DOM API calls (`createElement`, `textContent`) instead of string concatenation + `innerHTML`. At minimum, wrap `entry.opportunity` in `esc()`.

### Honest assessment of M4 completion

M4 is marked done, but the security review found three issues that contradict the done criterion ("the API requires auth outside of local environments"):

1. The default token `sentinel-secure-token` is published in the README, baked into the binary, baked into the Helm chart, and baked into the JS bundle. A bare `helm install` with no value overrides is "auth-enabled" in name only.
2. `/health` exposes internal IPs and service addresses to unauthenticated clients.
3. DOMPurify was removed in the same milestone that was supposed to harden security â€” that's a regression.

These aren't blockers for M5 feature work, but they should be patched before any public deployment or before the M6 "README reflecting real project state" goal.

### M5 â€” What to watch out for

**The `dashboard.js` file is getting large.** At ~1,900 lines it's already hard to navigate, and M5 ("LLM narrative enrichment", "automatic runbooks", "degraded mode") will add more rendering logic. If you don't modularize before M5 closes, M6's "revised dashboard UX" goal becomes a rewrite, not a polish. Consider splitting the JS into logical embedded modules (e.g. `drawer-nodes.js`, `drawer-finops.js`) before the LLM integration piles on.

**The `/incident` â†’ `/api/incidents` wiring is well-designed on paper.** Consuming the deterministic endpoint first, then optionally enriching with an LLM narrative, is the correct architecture. The main risk in M5 is that the LLM narrative path gets built before the degraded-mode fallback is robust â€” prioritize the degraded mode first.

**The harness (`validador_saida.py`) covers the output side but not the input side.** When the LLM layer starts generating Kubernetes remediation commands (not just reports), the current block list (`rm -rf`, `kubectl delete`, `DROP TABLE`) will need to evolve. Plan this before M5 generates runbooks with executable steps.

**The honeycomb node health drawer is functionally solid now** (Datadog-style pod list per node, mock-node behavior, click-to-focus). That M5 backlog item is mostly done from the UI side. What's missing for M5 completion is the LLM enrichment of incidents â€” the UI is ready to receive a `narrative` field on `Incident` objects if the backend adds it.
