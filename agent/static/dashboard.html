Efficiency Grades
Score = actual usage ÷ requested resources × 100
A85–100% — perfectly provisioned. Actual usage close to requests.
B70–84% — well provisioned. Minor headroom but acceptable.
C50–69% — moderate waste. Consider right-sizing requests.
D30–49% — significant waste. Priority for optimization.
F<30% — critical waste. Immediate action recommended.
—UNMANAGED — no CPU/memory requests set. No basis for score.
FinOps — metric glossary
Reference price: ~$0.048 / core-hour • ~$12.50 / month per 1m CPU
Budget ($)Cost of requested (reserved) resources — what you committed to pay regardless of actual use. Lower is better.
Actual ($)Real cost of actual CPU/memory consumption — what you actually used. This is the baseline.
Waste ($)Budget − Actual — resources reserved but not consumed. This is your optimization opportunity.
Waste %Waste ÷ Budget × 100 — efficiency ratio. <20% is lean; >40% is critical.
Proj. BudgetForecast of committed spend for the next time window, based on current request patterns.
Proj. UsageForecast of actual consumption — expected real cost based on usage trends.
Proj. WasteForecast of waste for the next period — use this to plan right-sizing actions.
±1.5σ95% confidence band — statistical margin. Real value likely falls within this range.
